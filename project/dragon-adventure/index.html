Dragon Adventure

Dragon Adventure

Start: October 15, 2021

Platforms: Android, IOS, Linux

Project Info

Project Length: 3 Months

Project Type: Personal

Responsibilities: Gameplay Programming

Engine and tools: Unity

Languages: C#

Description:

My first Unity project.
I started this project to deepen my understanding of the Unity Engine.
I worked on it for over three months, covering all stages—from designing and brainstorming ideas, creating assets, learning Unity, to programming the entire system.

Game inspired by Flappy Bird.
You control a baby dragon flying through different lands, taking down enemies and uncovering hidden truths behind the journey.

Controls:

  • Tap left side to make the dragon jump/fly upward.

  • Tap right side to shoot projectiles.

  • Defeat monsters to charge your rage gauge.

  • When mana and rage are full, tap the Skill button to unleash a powerful ability.

What I Do:

  • Brainstorm and develop game ideas

  • Design and illustrate game assets

  • Create visual effects using particle systems

  • Design gameplay mechanics and systems

  • Program game logic

Animations

Description:

The 2D character is divided into several parts: Head (Horns, Eyes, Tongue, Upper Jaw, Lower Jaw), Wings, Legs, Arms, Body, and Tail. Animations are structured into 3 layers: Eye, Head, and Body.
  • Eye Layer: Controls blinking
  • Head Layer: Controls head rotation and mouth movement
  • Body Layer: Controls movement of legs, arms, wings, tail, and body
Uses Unity's built-in Animator and Animation system, allowing for condition-based management and smooth transitions between animation clips.

Parallax Background
Code Demo 
    
	public float parallaxEffect = 1; //Move speed same camera speed;
	// Start is called before the first frame update
	void Start()
	{
		_MainCamera = Camera.main;
		startPos = transform.position.x;
	}

	// Update is called once per frame
	void Update()
	{
		float distance = _MainCamera.transform.position.x * parallaxEffect;
		transform.position = new Vector3(startPos - distance, transform.position.y, transform.position.z);
	}

Code Explanation: Each individual layer — background, foreground, and playerground — is assigned a different parallaxEffect value.
As the camera moves, each layer scrolls at a different speed,
creating a sense of depth in the game.

Sprite Shape

Description:

I’m using Sprite Shape for the ground and upper boundary due to its smooth curvature.

For performance optimization, I use a single continuous Sprite Shape for the ground and another for the upper edge. I minimize the number of spline points, mark the objects as static, and disable collider updates.

Input Processing

Description:

For Unity Editor:

  • When the Space key is pressed, the character jumps by applying predefined forces and updating animation states.
  • When the left mouse button is held down, the character continuously fires (attack is triggered) until the mouse button is released.

This block handles input specifically for the Unity Editor:


		#if UNITY_EDITOR
		
		// Process Moving
		if (Input.GetKeyDown(KeyCode.Space))
		{
			isHoldingOnLeft = true;
			status = PlayerStatus.Normal;
			anim.SetBool("isIdle", false); // onGround = false;
		
			// tutorialManager.SetIndex(-1); // Passed step 1;
		
			// Set Force
			tempJumpForce = jumpForce;
			tempMoveForce = moveForce;
			rgbd2D.velocity = new Vector2(0, 0);
			Moving();
			tempJumpForce = additionalJumpForce;
			tempMoveForce = additionalMoveForce;
		}
		
		if (Input.GetKeyUp(KeyCode.Space))
		{
			isHoldingOnLeft = false;
			holdingDuration = 0f;
		}
		
		// Process left mouse clicked => Shooter
		if (EventSystem.current.IsPointerOverGameObject())
		{
			return;
		}
		
		if (Input.GetMouseButtonDown(0))
		{
			var clickPos = Input.mousePosition;
			if (attackInputEnabled)
			{
				emberSkillManager.TryAttack(clickPos, isDown: true);
			}
		}
		
		if (Input.GetMouseButtonUp(0))
		{
			var clickPos = Input.mousePosition;
			emberSkillManager.TryAttack(clickPos, isDown: false);
		}

For Mobile:

  • Touching the left half of the screen causes the character to move/jump.
  • Tapping repeatedly or holding on the right half of the screen allows the character to shoot continuously.

		#elif UNITY_ANDROID || UNITY_IOS
		
		if (Input.touchCount > 0)
		{
			Touch[] myTouches = Input.touches;
		
			foreach (Touch touch in myTouches)
			{
				// If touch on left side => Movement
				if (touch.position.x < Screen.width * 2 / 5)
				{
					if (touch.phase == TouchPhase.Began)
					{
						isHoldingOnLeft = true;
						status = PlayerStatus.Normal;
						anim.SetBool("isIdle", false); // onGround = false;
		
						// tutorialManager.SetIndex(-1); // Passed step 1
		
						// Set Force
						tempJumpForce = jumpForce;
						tempMoveForce = moveForce;
						rgbd2D.velocity = new Vector2(0, 0);
						Moving();
						tempJumpForce = additionalJumpForce;
						tempMoveForce = additionalMoveForce;
					}
		
					if (touch.phase == TouchPhase.Ended)
					{
						isHoldingOnLeft = false;
						holdingDuration = 0f;
					}
				}
				// If touch on right side => Shooting
				else
				{
					if (EventSystem.current.IsPointerOverGameObject(touch.fingerId))
					{
						return;
					}
		
					if (touch.phase == TouchPhase.Began)
					{
						emberSkillManager.TryAttack(touch.position, isDown: true);
					}
					else if (touch.phase == TouchPhase.Ended)
					{
						emberSkillManager.TryAttack(touch.position, isDown: false);
					}
				}
			}
		}
		
		#endif

I learned how various parameters influence the behavior of particles—such as their direction, shape, speed, color, brightness, and form. This helped me understand how to fine-tune effects for different gameplay scenarios.
